1 INTRODUCTION

A modern sort of attack used for phishing is called a "Browser in the middle attack" (BITM). It involves an attacker
intercepting and changing the communication between a user's web browser and a website.1­3 The communication route
between a user's browser and a targeted website is intercepted and monitored as part of a BITM attack, also known as
a Man in the Middle assault unique to browsers1,3 as shown in Figure 1. The attacker effectively served as a proxy by
placing himself between the user's browser and the target website. Unknowingly sharing information with the attacker,
who then forwards it to the desired website, the user communicates with the attacker.4

    BITM attack tricks its victim into believing they are browsing normally and that the connection is secure.1 The vic-
tim's movements while gazing out the open window are actually captured on the adversary's computer. The attacker has
accessed these victim-authenticated sessions. The adversary has the ability to capture all input data, including usernames
and passwords, and can alter the content that is shown to the victim or may be this attack will be used as a ransomware
attack. Browser in the middle attacks can be used to compromise multifactor authentication systems that rely on user
input and avoid the hardware-based secret exchange.1,3

1.1 History and statistics

The BITM attack can be considered as a modern phishing technique. Researchers from corporations and academic insti-
tutions are equally interested in phishing and BITM scams because they pose a severe risk to online privacy and security.
Firewalls and other encryption tools are not effective to stop phishing and BITM attack.5,6

2 WORKING OF BITM ATTACK

The working of attack is divided into seven phases which are followed by the attacker to perform the attack. An outline
of a BITM attack's potential execution in the actual world is provided in Figure 3.

2.1 Initial stage

The attacker sends the malicious link using e-mail to the users and after the user opens the link the attacker configures
their system (by the help of malicious link) to eavesdrop on network traffic that passes between users and the target web-
site. This can be accomplished in a number of ways and other ways includes compromising the user's network connection,
a router, or by setting up a rogue access point.

2.2 Blocking the communication channel

A user's browser sends a request to the website's server whenever they attempt to connect to that website. The attacker
intercepts this request, typically using techniques like DNS or address resolution protocol (ARP) spoofing. As a result, the
attacker has the opportunity to block access to the website and the user.

2.3 Making a fraudulent certification

For the target website, the attacker makes a false SSL/TLS certificate. A certificate authority (CA) that the user's browser
trusts has signed the certificate. This step is essential for creating a secure HTTPS connection without causing the user's
browser to issue any alerts.

2.4 Transmitting traffic

The attacker decrypts the traffic while forwarding the user's request to the trustworthy website using the phony certificate.
By doing so, they are able to examine and alter the data that is being sent between the user and the website.

2.5 Data manipulation

The intercepted traffic is now entirely visible to and under the attacker's control. They have the power to alter the website's
content, add malicious code or scripts, steal private information, redirect users to another website, or do any of these
things.

2.6 Returning the modified data

The attacker re-encrypts the modified traffic using the real SSL/TLS certificate of the real website. This step ensures that
the updated data is regarded as reliable by the user's browser.

2.7 User interaction

The attacker modifies the response and sends it to the user's browser, which displays it as if it were the genuine response
from the website. The user's accidental interaction with the updated content could weaken the system even more or open
the door to unauthorized actions.

    A successful BITM attack requires the ability to change SSL/TLS certificates, access to the network infrastructure,
and technical understanding. To safeguard oneself from such attacks, users should make sure their devices and browsers
are up to date, rely on secure network connections (such as HTTPS), and exercise caution when accessing the internet,
especially when entering personal information.

3.2.1 Infection using malware

The attacker uses a variety of methods, including phishing emails, attachments, pop-up ads on websites, and infected

begin to run in the background and infect the user's web browser.3,18

3.2.2 Manipulation of user browser

The malware gains the ability to change how content is presented in the browser. It has the ability to alter web pages,
add new scripts, or create pop-up ads which will further used for the attack. With this skill, an attacker can trick people
into giving up sensitive data such as login passwords and financial information and sometimes personal information like
photos, videos, and so on.2,18

3.2.3 Information capture

This malware can capture and send the user's input including credentials information, sensitive information about users.
This information can be further used for many purposes like identity theft, fraud, and many more purposes.17­19

3.3 Browser in the middle attack

The main entities in BITM attack are attacker, victim and its browser, and the targeted website.

3.3.1 Attacker

Attacker is the computer criminal and its aim is to violating the CIA triad (Confidentiality, Integrity, Availability) of the
user. In this attack, attacker use to setup and control the user's browser and use any malicious server which will act as a
transparent browser to the user.

3.3.2 Victim and its browser

The end result of this attack is phishing only. The information of user's browser and all the personal information of the
user is compromised with the attacker.

3.3.3 The targeted website

A web application is targeted which will help attacker to provide information of the user. The attacker used to modify the
content of the targeted website to theft the information of the user.3,18

4.1 Exploiting browser plugins or extensions

Attackers may create dangerous plugins or extensions that users install without their knowledge. By inserting code into
online pages, altering network requests, or altering browser behavior, an attacker can use these extensions to intercept or
tamper with interactions between the browser and websites.1

    In some cases, popular browser extensions that are considered safe are sold or hijacked by shady organizations.
Malicious groups that employ such extensions set up updates that can turn seemingly harmless extensions into mal-
ware. Compromised extensions connect browsers to command-and-control architectures to filter out sensitive data of
unsuspecting users, exposing them to further risks and this risk leads to BITM attack.

4.2 Wi-Fi packet sniffing

An attacker within range of the target device can capture and examine network traffic if it is linked to an unprotected Wi-Fi
network. The attacker can access confidential information such as session cookies or login information by sniffing the
packets.15 Wi-Fi sniffers are packet sniffers or network analyzers' that collect packet data on wireless networks. Wireless
sniffer solutions are intended to capture and analyzer wireless network data in order to obtain insight into what is going
on in user's network at all times. Wi-Fi sniffers are sometimes referred to as Wi-Fi packet sniffers and Wi-Fi network
sniffers. Wireshark, Nmap, SolarWinds npm, and other sniffing tools are commonly used for sniffing Wi-Fi packets.20
Packet analysis using Wireshark is shown in Figure 7. Attackers can intercept and examine browser traffic using Wi-Fi
packet sniffing, which makes it easier to carry out BITM attacks by obtaining session tokens, private information, and the
ability to manipulate content. Network procedures and secure protocols like HTTPS are crucial for prevention.

4.3 MITM proxies

A software program known as a proxy server serves as a middle man between a server and end users, including computers
and mobile devices. When a user requests a web page or data, it sends the request to the base server, which hosts resources
including files, web pages, and videos. The destination address of the answer is provided by the server. This common
practice of requesting web pages or receiving responses from the web server is unsafe because a third party may intercept
user's IP address and attack their client device, PC, or terminal.4

4.5 DNS spoofing

By winning the trust of the victim (the target user), hackers can distribute malware and steal information like passwords
and PINs that are saved on the victim's device. This type of attack is known as spoofing. The basic goal of spoofing is to
psychologically manipulate the victim. A sort of cyberattack known as DNS spoofing involves a hacker intercepting DNS
request and responding with bogus data id shown in the Figure 9. If a user attempt to visit google.com, for instance, a
hacker may intercept user's DNS request and reply with a phony IP address that points to a malicious website. Linking a
legitimate website's domain name to a different IP address under the attacker's control requires altering a target device's
or network's DNS cache. When a user attempts to access a website, their browser is instead directed to the attacker's
server, enabling communication interception and manipulation.15

    DNS spoofing is typically used to route consumers to a phony website that mimics the legitimate one. A phishing
attempt like this one can be used to acquire sensitive data including login credentials, credit card details, and more. In
some situations, DNS spoofing can also be used to send people to malicious websites that can infect their computers and
steal information or cause other problems. In Figure 8, the client/user initially requests the DNS server, but the attacker
intercepts the connection and injects fake DNS entries into the DNS server. As a result, the connection is resolved to a
fake website, which is then used for the BITM attack.

    With the use of bogus DNS replies, DNS spoofing in a BITM attack enables content modification, phishing, and the
capture of sensitive data, all of which can lead an attacker's browser to a hostile website. Inadvertent access and possible
security breaches result from this undermining the integrity of the domain resolution procedure.

5 TAXONOMY OF BITM PREVENTION TECHNIQUES

Phishing scams have been used all over the internet to trick unsuspecting victims into revealing personal information like
bank account details, social security numbers, and so on. Additionally, cybercriminals have improved their disguising
skills. Sometimes the voices of user's coworkers, bank, or even user's government serve as a cover for these scams. There-
fore, this section presents detailed description about some of the prevention techniques which will help us to prevent the

9 CONCLUSION AND FUTURE SCOPE

This paper has explored various facts of phishing through BITM and its utilization for phishing, along with its implications
for data and system security. Additionally, it has delved into preventive techniques that can help avert or mitigate the

cognizant of common signs and risks of phishing and adopt good security practices such as verifying the sender's address,
confirming the URL's domain, checking for spelling or grammar errors, refraining from clicking on suspicious links or
attachments, and reporting phishing attempts.

    Further future scope is to prepare a detection system which is capable of detecting BITM attack. Detection system
should be detecting the attack as well as mitigate the attack so that the impact of the attack would be minimize. The
browser fingerprint should be there in detection system which will monitor the browser for any malicious activity and
able to detect it.
